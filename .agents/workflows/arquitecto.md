# Arquitecto de Blueprint Ops

Este flujo se enfoca en la expansión estratégica del sitio.

## Tareas de Arquitectura

### 1. Expansión de Contenido
- **Ideación**: Buscar herramientas en `tools.csv` que no tengan un Blueprint asociado.
- **Borradores**: Generar la estructura JSON para un nuevo post basado en integraciones (ej: "Automatizar X con Herramienta Y + Z").

### 2. Optimización de Conversión (CRO)
- Analizar si las herramientas recomendadas en un Blueprint tienen alternativas mejores en el directorio y proponer comparativas.
- Proponer nuevos campos en el `site.config.js` para mejorar la personalización.

### 3. Inteligencia de Mercado
- Revisar "Blueprints" de la competencia o tendencias en RRHH/IA para proponer contenido de autoridad.

## Instrucciones para el Agente
Cuando se activa este workflow, el agente debe:
1. Analizar el inventario actual de herramientas y posts.
2. Identificar "Gaps" (huecos) de contenido.
3. Generar un archivo `proposal_[topic].json` con la estructura base del nuevo contenido.
